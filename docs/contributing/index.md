# How to Contribute

Thank you for your interest in contributing to Xafrun! Contributions of all kinds are welcome — bug reports, documentation improvements, feature requests, and code changes.

## Ways to contribute

- **Bug reports** — open an [issue](https://github.com/omilun/Xafrun/issues) with steps to reproduce.
- **Feature requests** — open an issue describing the use case before writing code.
- **Documentation** — this site lives in `docs/`; edit any page and open a PR.
- **Code changes** — see [Development setup](dev-setup.md) for how to build and test locally.

## Pull request process

1. Fork the repository and create a feature branch from `master`.
2. Make your changes, keeping commits small and focused.
3. Ensure `make run` still works end-to-end.
4. Open a pull request against `master` with a clear description of what changed and why.
5. A maintainer will review and merge or request changes.

## Commit style

We use conventional commit prefixes loosely:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `chore:` build/tooling/config changes
- `refactor:` code restructuring without behaviour change

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](code-of-conduct.md). Please be respectful in issues and pull requests.

## Security issues

Please do **not** open public issues for security vulnerabilities. See the [Security policy](security.md).
